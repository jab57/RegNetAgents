**LLM-Powered Multi-Agent Framework for Gene Regulatory Analysis**

RegNetAgents is an AI-powered multi-agent framework that provides comprehensive gene regulatory network analysis through Claude Desktop. Built on **pre-computed gene regulatory networks** derived from single-cell RNA-seq data, and powered by LangGraph workflow orchestration, it deploys four specialized LLM-powered domain agents (cancer biology, drug discovery, clinical relevance, and systems biology) that generate scientific insights using local Ollama inference.

**Analyze 10 different cell types** including immune cells, blood cells, and epithelial tissue to understand gene regulation, discover biomarkers, and explore disease mechanismsâ€”with AI-generated rationales and interpretations.

**Network Data:**
- **Primary Source**: GREmLN Foundation Model (Zhang et al. 2025, bioRxiv 2025.07.03.663009)
- **Download Location**: [GREmLN Quickstart Tutorial](https://virtualcellmodels.cziscience.com/quickstart/gremln-quickstart) (networks via Google Drive)

- **Networks Used**: 10 cell-type-specific networks (500K+ cells subset from tutorial datasets)
- **Processing**: ARACNe algorithm via GREmLN team preprocessing (statistical mutual information method)
- **Development**: CZ Biohub NY / Columbia University (Califano Lab)
- **Format**: TSV files (network.tsv) converted to NetworkX-compatible pickle caches (network_index.pkl)
- **GREmLN GitHub**: https://github.com/czi-ai/GREmLN

**What "RegNetAgents" means:**
- This framework focuses on **Regulatory Network** analysis using AI **Agents**
- Uses pre-computed regulatory networks from GREmLN (generated by the ARACNe algorithm)
- Network analysis is performed using NetworkX (graph theory), not machine learning inference
- The name reflects the core functionality: multi-agent analysis of gene regulatory networks

## Troubleshooting

### "Network index file missing" error

**Problem:** MCP server fails to load a cell type.

**Solution:**
```bash
# Check which cell types have cache files
ls models/networks/*/network_index.pkl

# If missing, generate cache:
python scripts/build_network_cache.py --all
```

### Claude Desktop doesn't show RegNetAgents tools

**Problem:** Tools don't appear in Claude Desktop.

**Solutions:**
1. Verify the config path is correct in `claude_desktop_config.json`
2. Use **absolute path**, not relative (e.g., `C:\\Users\\...` not `./`)
3. Use double backslashes on Windows (`\\` not `\`)
4. Restart Claude Desktop completely (quit and reopen)
5. Check Claude Desktop logs for errors

### "Module not found" errors

**Problem:** Python import errors when starting server.

**Solution:**
```bash
# Ensure virtual environment is activated
# Windows:
env\Scripts\activate

# Linux/Mac:
source env/bin/activate

# Reinstall dependencies
pip install -r requirements.txt
```

### Analysis returns empty results

**Problem:** Gene analysis completes but shows no regulators/targets.

**Possible causes:**
1. Gene not in network for that cell type (normal - try different cell type)
2. Gene symbol not recognized (try Ensembl ID like ENSG00000141510)
3. Network cache corrupted (regenerate with `build_network_cache.py`)

### Reactome pathway analysis fails

**Problem:** Pathway enrichment section shows errors.

**Solution:** Requires internet connection. Check network connectivity or firewall settings.

## Example Use Cases (No Biology Degree Required)

### Cancer Research
**Question**: "What regulates the APC gene in colon cells?"
- **Why it matters**: APC mutations cause colorectal cancer - it's one of the most commonly mutated genes in colon tumors
- **What you'll learn**: Which genes control APC, what pathways it participates in, and potential therapeutic targets
- **Try it**: `comprehensive_gene_analysis` with gene="APC", cell_type="epithelial_cell"

### Drug Discovery
**Question**: "What genes does BRCA1 regulate in breast tissue?"
- **Why it matters**: BRCA1 mutations dramatically increase breast cancer risk (60-80% lifetime risk)
- **What you'll learn**: Downstream targets for potential drug interventions, pathway disruptions, druggable proteins
- **Try it**: `comprehensive_gene_analysis` with gene="BRCA1", cell_type="epithelial_cell"

### Therapeutic Target Prioritization
**Question**: "What happens if I inhibit TP53 upstream regulators?"
- **Why it matters**: Identifies which regulators have the biggest impact when targeted by drugs or therapies
- **What you'll learn**: Top therapeutic targets ranked by network centrality metrics (PageRank, out-degree centrality), cascade effects, druggability assessment
- **Network metrics**: Uses established centrality measures from computational biology literature (PageRank, degree centrality, out-degree centrality)
- **Evidence-based ranking**: PageRank is the best predictor of successful drug targets according to Mora & Donaldson (2021)
- **Try it**: `comprehensive_gene_analysis` with gene="TP53" - therapeutic target prioritization runs automatically for genes with >5 regulators
- **Example output**: "WWTR1 (PageRank 0.47) would reduce TP53 regulation by 14.3% and affect 293 downstream genes"

**Important Limitation**: Perturbation analysis is topology-based and ranks regulators for experimental investigation. It does **NOT** predict gene expression changes or model dynamic regulatory responses. Results prioritize candidates for validation, not expression predictions.

### Understanding Disease Mechanisms
**Question**: "How does TP53 behave differently across cell types?"
- **Why it matters**: TP53 is mutated in over 50% of all cancers - it's the most important tumor suppressor
- **What you'll learn**: Cell-specific regulatory networks, why some tissues are more cancer-prone than others
- **Try it**: `multi_gene_analysis` or use cross-cell comparison features

### Immune System Research
**Question**: "What pathways are enriched for IL6 in immune cells?"
- **Why it matters**: IL6 is a key inflammatory signal involved in autoimmune diseases and COVID-19 severity
- **What you'll learn**: Biological pathways, potential drug targets for reducing inflammation
- **Try it**: `pathway_focused_analysis` with gene="IL6", cell_type="cd14_monocytes"

**Popular genes to explore**:
- Cancer: TP53, BRCA1, BRCA2, APC, KRAS, MYC, PTEN, RB1
- Immune system: IL6, TNF, IFNG, IL10, CD4, CD8A
- Blood cells: HBB (erythrocytes), CD4 (T cells), CD8A (T cells)
- Inflammation: IL6, TNF, IFNG (monocytes)

## Terminology Guide

**Core Concepts**:
- **Gene**: A section of DNA that contains instructions for making a protein (e.g., "TP53", "BRCA1")
- **Regulator**: A gene that controls whether another gene is turned "on" or "off"
- **Target**: A gene that is being controlled by a regulator
- **Gene Symbol**: Short name for a gene (e.g., "APC") - what humans use
- **Ensembl ID**: Database identifier (e.g., "ENSG00000134982") - what computers use

**Network Terms**:
- **Regulatory Network**: Map showing which genes control other genes in a specific cell type
- **Hub Regulator**: A gene that controls many other genes (>20 targets - like a master switch)
- **Heavily Regulated**: A gene that is controlled by many regulators (>15 regulators - complex control)
- **Weakly Regulated**: A gene with few regulators and no downstream targets (simple endpoint)

**Analysis Terms**:
- **Pathway**: A group of genes working together for a biological process (e.g., "Wnt signaling", "Cell cycle")
- **Enrichment**: Statistical test showing if certain pathways are over-represented
- **p-value**: Probability the result happened by chance (lower = more confident)
- **FDR (False Discovery Rate)**: Corrected p-value accounting for multiple tests (< 0.05 = statistically significant)
- **Therapeutic Target Prioritization**: Network-based ranking of regulators by centrality metrics (topology-based target prioritization, not dynamic expression simulation)

**Network Centrality Metrics** (for ranking therapeutic targets):
- **PageRank**: Google's algorithm adapted for biology - measures connection quality (best predictor of drug targets)
- **Degree Centrality**: How many connections a gene has (total incoming + outgoing) - measures overall connectivity
- **Out-Degree Centrality**: How many downstream targets a gene regulates - identifies hub regulators and cascade breadth
- **Regulatory Loss**: Percentage of target gene's regulation lost when a regulator is inhibited (for interpretation, not ranking)

**Cell Types** (10 available):
- **Epithelial cells**: Barrier tissues (skin, lung, intestine) - where most cancers start
- **CD14/CD16 Monocytes**: Immune cells that fight infections
- **CD4/CD8 T cells**: Adaptive immune cells that attack pathogens and cancer
- **B cells (CD20)**: Produce antibodies for immune defense
- **NK/NKT cells**: Natural killer cells that attack infected/cancer cells
- **Erythrocytes**: Red blood cells that carry oxygen
- **Dendritic cells**: Activate immune responses

**Don't worry if you don't know all these terms** - the MCP server provides explanations in its outputs!

## Features

### Current Capabilities
- **Network Analysis**: Analyze gene positions in regulatory networks
- **Therapeutic Target Prioritization**: Rank therapeutic targets using network centrality metrics (PageRank, out-degree centrality) from computational biology literature
- **Cell-Type Specific**: Support for 10 cell types with pre-computed networks
- **LLM-Powered Domain Analysis**: 4 specialized AI agents (cancer, drug development, clinical relevance, systems biology)
  - **Primary mode**: LLM-generated insights with scientific rationales (Ollama/llama3.1:8b)
  - **Biological context**: Gene functional descriptions from NCBI/UniProt databases provide validated biological information to LLM agents
  - **Graceful fallback**: Rule-based heuristics if LLM unavailable (reliability feature)
  - Each analysis result includes `llm_powered: true/false` flag for transparency
- **Pathway Enrichment**: Reactome API integration with p-values and FDR
- **Statistical Validation**: Quantitative pathway enrichment scores
- **Cross-Cell Comparison**: Gene behavior across tissue types
- **Network Statistics**: Detailed network information

### MCP Tools Available

1. **comprehensive_gene_analysis**
   - Complete workflow-driven gene analysis with domain expertise
   - Parameters: gene, cell_type, analysis_depth
   - Returns: Network analysis, regulators, targets, Reactome pathways (with p-values/FDR), cross-cell, domain insights

2. **multi_gene_analysis**
   - Parallel processing of multiple genes
   - Parameters: genes[], cell_type, analysis_depth
   - Returns: Batch analysis results

3. **pathway_focused_analysis**
   - Pathway-specific analysis using Reactome enrichment
   - Parameters: pathway, genes[], cell_type
   - Returns: Statistical pathway enrichment with p-values and FDR

4. **workflow_status**
   - Real-time workflow monitoring
   - Returns: Execution status, performance metrics

5. **workflow_insights**
   - Advanced workflow analytics
   - Parameters: analysis_type (performance, routing, completeness)

6. **create_analysis_report**
   - Generate comprehensive reports
   - Parameters: format (summary, detailed, scientific)

## Performance

**Measured via comprehensive timing tests (with pre-computed PageRank):**

**Rule-Based Mode** (fast, deterministic):
- **Focused single gene**: 0.68 sec (network, regulators, targets, complete therapeutic target prioritization)
- **Comprehensive single gene**: 0.60 sec (focused + Reactome pathway enrichment)
- **Multi-gene (5 genes)**: 15.49 sec (parallel execution, complete perturbation of all 99 regulators)
- **Cross-cell comparison**: <0.01 sec (instant lookups from pre-computed indices)

**LLM-Powered Mode** (AI-generated insights with rationales using Ollama/llama3.1:8b):
- **Comprehensive single gene**: ~15 sec (includes 4 domain agents with scientific rationales)
- **Multi-gene (5 genes)**: ~62 sec (parallel execution + LLM insights for all genes)

**API Latency**:
- **Reactome API**: 0.3-0.5 sec per query (depends on internet speed)
- **Ollama LLM**: 3-4 sec per agent call (local inference, no API costs)

**Optimization**: PageRank pre-computed in network cache (Version 2) provides instant therapeutic target prioritization.
